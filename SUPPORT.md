# Support

This document explains support scope and issue routing for `Lkkisme/Cap`.

## Scope

This fork is maintained for Chinese-language and Windows-focused use cases around the open source Cap screen recorder.

Supported topics:

- Windows installation, launch, update, and release artifact problems.
- Chinese-language documentation or user workflow issues.
- Bugs that reproduce in this fork.
- Build and development setup for this repository.
- Privacy, security, signing, SmartScreen, WDSI, WinGet, and release trust questions related to this fork.

Out of scope:

- Paid support commitments or guaranteed response times.
- Official upstream Cap support.
- Issues that only affect `CapSoftware/Cap` and not this fork.
- Requests to bypass authentication, subscription controls, organization policies, SmartScreen, or endpoint protection.
- Private troubleshooting that requires sharing secrets, credentials, private recordings, or confidential company data in public.

## Where to Ask

- Reproducible bugs: open a GitHub issue using the bug report template.
- Windows installer, signing, SmartScreen, updater, or release package problems: use the Windows release problem template.
- Feature requests: use the feature request template.
- Security vulnerabilities: use GitHub private vulnerability reporting at https://github.com/Lkkisme/Cap/security/advisories/new.
- Upstream-only Cap issues: use https://github.com/CapSoftware/Cap.

## Response Expectations

This project is maintained on a best-effort basis.

Expected triage:

- Security issues: handled privately and prioritized based on impact.
- Release blockers: prioritized before routine improvements.
- Reproducible Windows install or launch problems: prioritized when enough details are provided.
- General feature requests: reviewed when maintainer time allows.
- Incomplete reports: may be closed if the requested information is not provided.

No guaranteed service-level agreement is promised.

## Privacy

Do not post:

- Tokens, API keys, cookies, credentials, session IDs, or signing secrets.
- Private screen recordings, microphone audio, camera video, or customer files.
- Logs containing emails, account IDs, bucket names, private paths, or company information unless they are redacted.

If a report requires sensitive details, ask for a private contact path first.
