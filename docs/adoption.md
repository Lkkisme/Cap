# Adoption

This document tracks adoption evidence for `Lkkisme/Cap` only.

`Lkkisme/Cap` is a public unofficial Chinese Windows-focused fork of `CapSoftware/Cap`. The upstream project has substantial public visibility, including about 19.7k GitHub stars and 1.6k forks as background context checked on 2026-06-21. Those upstream numbers are not adoption metrics for this fork.

## Current Status

This fork is early and currently has low directly observed adoption.

No inflated usage claims are made here. If adoption data has not been independently verified, it should be recorded as unknown rather than estimated.

## Why This Still Matters

The fork's current value is not broad adoption. Its current value is focused maintenance for a specific distribution gap:

- Chinese-language Windows users may need direct binaries instead of developer toolchains.
- Company-managed Windows computers may block app stores, unsigned binaries, or untrusted download paths.
- Screen recording apps require careful privacy, permission, release integrity, and support documentation.
- The upstream Cap project has ecosystem importance, while this fork documents a downstream path for Windows release trust and Chinese user support.

This section should not be used to claim broad usage. It explains why the fork can still be useful while adoption evidence is still small.

## What Counts as Adoption Evidence

Valid evidence includes:

- Public GitHub stars, forks, watchers, issues, pull requests, and release downloads for this fork.
- Public discussion threads that clearly refer to this fork.
- User reports that are public or approved for documentation.
- External contributions that are visible in the repository.
- Reproducible download or usage statistics from maintainer-controlled release channels.

## What Does Not Count

The following should not be counted as adoption of this fork:

- GitHub stars, forks, or downloads from `CapSoftware/Cap`.
- General popularity of the upstream Cap project.
- Private messages that cannot be cited or summarized with permission.
- Estimated users without a source.
- Aspirational statements about future users.
- Automated traffic, bot activity, or repository views without context.

## Update Process

When updating this document:

1. Record the exact metric value.
2. Record the source.
3. Record the date checked.
4. Keep upstream and fork metrics separate.
5. Do not round numbers upward.
6. Do not infer active users from stars, forks, views, or clones.
7. Remove or correct any metric that cannot be verified later.

## Notes for Reviewers

This fork is early and currently has low direct adoption evidence. Its value should be evaluated through maintenance clarity, honest scope, Windows-focused usefulness, release trust work, and the ability to stay aligned with upstream, not through exaggerated usage claims.
