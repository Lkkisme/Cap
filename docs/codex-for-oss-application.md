## Human-Only Fields and Actions

The repository owner must fill these personally:

- First name.
- Last name.
- Email.
- OpenAI organization ID.
- Any required consent, terms, CAPTCHA, or account confirmation.
- Any field asking for personal identity, affiliation, or legal representation.

Do not infer the OpenAI organization ID from the repository. Use the value shown in the owner's OpenAI platform account.

## Role Selection Guidance

If the form asks about the submitted repository, the truthful role is:

`Primary maintainer of Lkkisme/Cap.`

If the form asks about upstream `CapSoftware/Cap`, the truthful answer is:

`Not a maintainer of upstream CapSoftware/Cap.`

If the form only offers a primary or core maintainer role choice, select it only if the submitted repository URL is `https://github.com/Lkkisme/Cap`, then clarify in a free-text field that this is for the fork.

## Risks and What Not to Claim

Do not claim:

- That the applicant maintains upstream `CapSoftware/Cap`.
- That the applicant represents Cap Software, Inc.
- That `Lkkisme/Cap` has the same stars, forks, downloads, users, or reputation as upstream Cap.
- That existing Windows release assets are already signed or SmartScreen-clean.
- That SignPath approval is complete unless it has actually been approved.
- That OpenAI credits will be used for private client work, paid proprietary work, or unrelated repositories.
- That the fork has more governance authority than it actually has.
- That the fork changes privacy behavior beyond what the public privacy policy states.
