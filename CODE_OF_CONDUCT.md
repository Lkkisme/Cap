# Code of Conduct

## Purpose

This repository is a public open source project. The goal of this code of conduct is to keep issue triage, pull request review, security coordination, and user support respectful and useful.

## Expected Behavior

Participants are expected to:

- Be respectful and constructive.
- Keep discussions focused on the project and its users.
- Assume good faith when asking for clarification.
- Share technical evidence when reporting bugs.
- Respect privacy when sharing logs, screenshots, recordings, or user data.
- Follow coordinated disclosure for security issues.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, threats, insults, or personal attacks.
- Publishing private information without consent.
- Posting secrets, credentials, private recordings, or sensitive logs.
- Repeatedly derailing issue or pull request discussions.
- Misrepresenting this fork as the official upstream `CapSoftware/Cap` project.
- Encouraging bypass of authentication, subscription controls, organization security policies, or endpoint protection.

## Scope

This code of conduct applies to:

- GitHub issues and pull requests.
- Discussions related to this repository.
- Security reports and release coordination.
- Any project-maintainer communication channel used for this fork.

## Enforcement

The maintainer may remove comments, lock threads, close issues, reject pull requests, or block users who repeatedly violate this code of conduct.

Enforcement should be proportionate, documented when appropriate, and focused on protecting contributors and users.

## Reporting

Report code of conduct concerns to Lkkisme through GitHub. If the report includes private information, contact the maintainer with a minimal public message asking for a private channel instead of posting details publicly.

Security vulnerabilities should follow `SECURITY.md` instead of being posted publicly.
